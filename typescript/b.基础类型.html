    • null: 此处有值，但是个空值
    • undefined: 这里没有值

# void

  • js
<a href="javascript:void(0)">清除缓存</a>
1

javascript 中

void(0)
等价于
void 0
(即
void expression
的语法) void 操作符会执行后面跟着的表达式并返回一个 undefined, 如你可以使用它来执行一个立即执行的函数

void (function life() {
  console.log('invoked');
})();
1
2
3

能这么做时因为， void 操作符强制将后面的函数声明 转化成了表达式， 因此整体相当于

void( (function life(){})() )
.

  • ts

ts 中也有 void， 与 js 不同的是，这里的 void 用于描述一个内部没有 return 的语句， 或者没有显示 return 一个值的函数的返回值, 如:

function fn1() {}
function fn2() return
function fn3() return undefined
1
2
3

fn1 和 fn2 返回值类型都被隐式推导为 void， 只有显示返回 undefined 的 fn3 才被推导为 undefined, 但实际执行后 fn1 fn2 fn3 返回值均是 undefined,

虽然 fn3 的返回值类型会被推导为 undefined， 但是仍然可以使用 void 类型进行标注， 可以理解为 void 表示一个空类型, 而 null、undefined 都是一个具有意义的实际类型（注意他们在 javascript 中的意义区别）。而 undefined ｜ null 能够被赋值给 void （关闭 strictNullChecks 配置情况下成立）

# 数组的类型标注

typescript 中有两种方式来声明一个数组类型

const arr1: string[] = [];
const arr2: Array<string> = [];
1
2

他们完全等价， 一般以前者为主。 数组是日常开发大量使用的数据结构， 在某些情况下， 使用 元祖(Tuple) 来代替数组要更加稳妥 比如一个数组中只存放固定长度的变量， 但我们进行了超出长度的访问

const arr3: string[] = ['c', 'q', 'c'];
console.log(arr3[666]);
1
2

这种情况是不符合预期的， 因为可以确定这个数组中只有三个成员， 这时候可以使用元祖类型进行类型标注

const arr4: [string, string, string] = ['c', 'q', 'c'];

console.log(arr4[666]); // error

// 可以声明不同类型的成员
const arr5: [string, boolean, number] = ['hello', false, 233];

// 标记可选的类型成员
const arr6: [string, string?, string?] = ['hi', , ,];
const arr7: [string, string?, string?] = ['hi'];
1

对于可选标记的成员， 在

--strictNullChecks
配置下会被视为 一个
string | undefined
的类型。 此时元祖的长度属性也会发生变化， 如上面的 arr6 其 长度类型为
1 | 2 | 3

type TupleLength = typeof arr6.length; // 1|2|3
1

也就是说， 这个元祖的长度可能是 1、2、3

# 元祖别名

如果觉着元祖可读性不好 😂 ， 比如

[string, number, number]
来说， 可能不知道他们具体代表什么， 不如使用对象形式, 而在
TypeScript 4.0
中, 有了具名元祖 Labeled Tuple Elementsopen in new window 的支持， 使得我们可以为元祖中的元素打上类似的属性标记

const arr8: [name: string, age: number, height: number] = ['cqc', 25, 170];
const arr9: [name: string, age: number, height?: number] = ['cqc', 25];
1
2

除了显示的越界访问， 还可能存在隐式的越界访问， 如通过解构赋值的形式

// 对于数组 没办法从类型层面检查里面有多少个元素
const arr1: string[] = [];
const [e1, e2, ...rest] = arr1;

// 对于元祖, 隐式的越界访问会抛出一个警告
const arr2: [name: string, age:: number] = ['cqc', 25];
const [name, age, other] = arr2; // err

# type 与 interface

一般比起 type（Type Alias， 类型别名），用 interface | class 的形式来描述对象、类的结构上比较推荐的, 而类型别名 用来 将一个函数签名、一组联合类型、一个工具类型等等抽离成一个完整独立的类型

# 总结

为了更好的区分 Object object {} 这三个具有迷惑性的类型，我们做下总结

  • 在任何情况下， 都

    不要使用 Object
    以及类似的装箱类型.

  • 当不确定某个变量的类型， 但能确定它不是原始类型， 可以使用 object. 但是更推荐进一步区分的

    • Record<string, unknown> or Record<string, any>， 表示对象
    • unknown[]
      or
      any[]
      , 表示数组
    • (...args: any[]) => any
      示函数

  • 同样要避免使用

    {}
    ,
    {}
    意味着 任何非
    null/undefined
    的值， 从这点来看， 使用它和使用
    any
    一样恶劣
